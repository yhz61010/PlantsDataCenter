# 植物知识库 · 结构化数据仓库设计

- 日期：2026-07-23
- 状态：待用户复核
- 方案：A（源 + 派生的单向管线）

## 1. 目标与背景

把现有 `knowledge/` 下 19 个 WPS xlsx（每文件一科、每工作表一物种、内嵌照片）
改造成一个**结构化数据仓库**：以一份规范的纯文本源为唯一真相，可派生出
JSON / SQLite / Markdown 多种格式，供程序读取、二次开发、AI/RAG 使用。

### 已确认的关键决定

| 决定项 | 选择 |
| --- | --- |
| 形态与用途 | 结构化数据仓库（非网站/文档站） |
| 消费方式 | 多种导出（JSON + Markdown + 数据库），来自单一源 |
| 真相源 | 纯文本源，每物种一个文件；xlsx 一次性导入后退役 |
| 图片 | 第一阶段忽略（不导出图片文件） |
| 源格式 | YAML |
| 管线架构 | 方案 A：一次性导入 → data/ 源 → 可重复导出 |

### 环境约束（已验证）

- 无 `pip`/`openpyxl`，但 **`PyYAML 6.0` 系统预装**（`/usr/lib/python3/dist-packages`）。
- 解析 xlsx、导出所需模块均在 Python 3.11 stdlib：`zipfile`、`xml.etree`、`sqlite3`、`json`。
- **整条管线零第三方安装、零额外依赖。**
- 有网络、`ensurepip` 可用（作为后备，正常流程用不到）。

## 3. YAML Schema（每物种）

### Schema 设计原则

- **固定字段与顺序**（全部必须出现）：`学名`、`中文名`、`俗名`、`异名`、`描述`、
  `分类系统`、`物种保护`、`分类信息`、`形态特征`、`生态习性`、`功用价值`、`植物志`、`元数据`。
  其中 `物种保护`/`分类信息`/`功用价值`/`植物志` 是本轮补充的区块（源数据里稀疏出现）。
- **缺失即补占位**：`俗名`/`异名` 缺失 → 字符串 `"无"`；`描述` 及 `分类系统`/`物种保护`/
  `分类信息`/`形态特征`/`生态习性`/`功用价值`/`植物志` 缺失 → 字符串 `"暂无数据"`。
- **区块值类型**：区块存在时为**开放子键的映射**（子键按各科实际标签保留，如
  识别要点/生活型/株/茎/枝/叶/花/果/种子/濒危等级/属种数/模式产地/经济价值/植物文化/产地/分布/生境/海拔/物候…）；
  区块缺失时为占位字符串。`植物志` 存在时为整段文本（尾部多行合并）。
- **两处无标签自由文本**：`异名` 之后、第一个区块之前的概述段 → `描述`；
  所有区块之后的尾部文本块（文献引证、变种描述、下级分类）→ `植物志`。
- **清洗规则**：去掉子键尾部 `：`、值尾部 `；`；异名去尾部 `\xa0(synonym)`；保留拼音声调与原文标点。
- **元数据块**：记录 `来源文件`、`来源工作表`，便于回溯核对。
- **写出参数**：`yaml.safe_dump(allow_unicode=True, sort_keys=False, default_flow_style=False, width=4096)`，
  中文不转义、字段顺序稳定、长行不折断。

## 4. 组件设计

### ① `scripts/import_xlsx.py`——一次性导入（xlsx → data/*.yaml）

- 用 `zipfile` 打开 xlsx，读 `xl/workbook.xml` 取工作表清单，**跳过 `WpsReserved_CellImgList`**。
- 读 `xl/sharedStrings.xml`（**按 `<si>` 合并其内 `<t>` run**）建字符串索引表；读每个
  `xl/worksheets/sheetN.xml`，把单元格按 **A/B/C 列坐标**还原成逐行网格（D 列的 `DISPIMG` 图片公式忽略）。
- **区块驱动的解析状态机**（区块标题在 A 列）：
  - 顶部字段：`学名 / 中文名`（A=标签、B=值）；`俗名`（B 值按 `、`/`,` 拆列表）；
    `异名`（A4 起，值在 B 列逐行，每个去尾部 `\xa0(synonym)`）。
  - `描述`：`异名` 之后、第一个区块标题之前的 C 列无标签段落。
  - 区块 `分类系统 / 物种保护 / 分类信息 / 形态特征 / 生态习性 / 功用价值`：
    A=区块标题，其后各行 B=子标签（尾 `：`）、C=值（尾 `；`），**子键开放式**。
  - `植物志`：进入区块之后出现的 C 列无标签文本（B 空），逐行合并成一段。
- **缺失即补占位**：解析完成后注入默认——`俗名`/`异名` 空 → `"无"`；`描述` 与六个区块、
  `植物志` 空 → `"暂无数据"`。
- **科目录名**从**源文件名**派生（`MX-木樨科.xlsx` → `木樨科`），不依赖可能缺失的 `分类系统.科`。
- **清洗**：去子标签尾 `：`、值尾 `；`；保留声调符号。
- **不静默丢弃**：遇到无法归类的结构保留原文到 `备注`，并打 `WARN`。
- **幂等**：重跑覆盖 `data/`；支持 `--dry-run` 只报差异。

### ② `scripts/validate.py`——校验 data/ 符合 schema

- 遍历 `data/**/*.yaml`，逐条检查：
  - **全部固定字段存在**（真实值或占位值均可）：学名/中文名/俗名/异名/描述/分类系统/
    物种保护/分类信息/形态特征/生态习性/功用价值/植物志/元数据。
  - `学名`、`中文名` 必须为真实非占位值；`学名` 形如二名法（属名首字母大写 + 种加词）。
  - `分类系统` 若为映射：六阶齐全，每阶匹配 `^[A-Z][A-Za-z ]+-.+\(.+\)$`；若为 `"暂无数据"` 则放行。
  - `俗名`/`异名` 为列表或字符串 `"无"`。
  - 文件名 == `中文名`。
- 退出码非 0 表示有错，便于将来接 CI / pre-commit。**纯报告，不改数据。**

### ③ `scripts/export.py`——派生导出（data → dist/*）

- 加载全部 `data/**/*.yaml` 到内存列表。
- 生成三种产物，支持 `--only json,md,sqlite` 选择子集：
  - `dist/plants.json`——物种数组，直接供程序读取。
  - `dist/plants.sqlite`——主表 `plant`（学名/中文名/科/属…）+ 关联表
    `synonym`、`common_name`、`morphology`，支持按科/属/性状查询。
  - `dist/md/<物种>.md`——frontmatter（学名/中文名/俗名/异名/分类系统）+ 正文
    （描述，及 物种保护/分类信息/形态特征/生态习性/功用价值/植物志 各小节），供 AI/RAG 与人读。

## 5. 数据流

```
knowledge/*.xlsx ──import_xlsx.py（一次性）──► data/<科>/<物种>.yaml（真相源，手工编辑）
                                                      │
                                            validate.py（校验，不改数据）
                                                      │
                              export.py（可重复）─────┼──► dist/plants.json
                                                      ├──► dist/plants.sqlite
                                                      └──► dist/md/<物种>.md
```

## 6. 错误处理

- 导入器对未知结构 `WARN` 并保留原文到 `备注`，绝不静默丢弃。
- 校验器汇总所有错误后以非 0 退出码结束，输出「文件:字段:问题」清单。
- 导出器对缺字段的记录跳过该字段而非崩溃，并在末尾汇总不完整记录数。

- 全部 **19 个 xlsx 均可读，共 42 个物种**（`MX-木樨科.xlsx` 重新导出后已修复）。

## 7. 验证方式

- 以 `KM-苦木科.xlsx`（最简样本·臭椿，只有分类系统/形态特征/生态习性）验证基本流程与占位补齐。
- 以 `D-豆科.xlsx` 的 `槐`（最全样本，含分类信息/功用价值/植物志/多子键）验证全字段解析。
- 以 `BH-百合科.xlsx` 的 `百合`（含物种保护、且无描述段）验证稀疏区块与 `描述→暂无数据`。
- 全量导入 19 个 xlsx（42 物种）后跑 `validate.py`，人工抽查 WARN 项与占位是否合理。

## 8. 范围与非目标（YAGNI）

- **不做**：图片提取/导出、网站/前端、双向同步回 xlsx、在线 API。
- **暂不做**：SQLite 若确认无需查询可后置；export 可先只出 JSON + Markdown。
- 第一阶段聚焦：xlsx → YAML 源 → 校验 → JSON/MD/(SQLite) 导出。
